	•	Mengelola product.
	•	Mengelola test batch.
	•	Mengelola creative.
	•	Upload gambar/video.
	•	Menandai winner.
	•	Membuat scale batch dari winner.
	•	Melihat lineage creative dalam canvas.
	•	Menginput metrics manual.
	•	Export data.
1.4 Architecture Style
Architecture yang direkomendasikan:
	•	Next.js app sebagai frontend dan app layer.
	•	Firebase Authentication untuk auth.
	•	Cloud Firestore sebagai primary database.
	•	Cloud Storage sebagai tempat file creative.
	•	Firestore Security Rules untuk data ownership.
	•	Storage Security Rules untuk file ownership dan validasi file.
	•	React Flow untuk lineage canvas.
	•	Explicit reference ID untuk relasi utama.
	•	Dedicated edges collection untuk graph relation.

6. Security Rules
6.1 Firestore Rule Principle
Semua document wajib punya userId.
User hanya boleh read/write document yang userId-nya sama dengan request.auth.uid.
6.2 Firestore Rules Draft
rules_version = '2';  service cloud.firestore {   match /databases/{database}/documents {     function isSignedIn() {       return request.auth != null;     }      function isOwner() {       return isSignedIn() && resource.data.userId == request.auth.uid;     }      function isCreatingOwnDoc() {       return isSignedIn() && request.resource.data.userId == request.auth.uid;     }      match /users/{userId} {       allow read, update, delete: if isSignedIn() && request.auth.uid == userId;       allow create: if isSignedIn() && request.auth.uid == userId;     }      match /products/{productId} {       allow read, update, delete: if isOwner();       allow create: if isCreatingOwnDoc();     }      match /testBatches/{batchId} {       allow read, update, delete: if isOwner();       allow create: if isCreatingOwnDoc();     }      match /creatives/{creativeId} {       allow read, update, delete: if isOwner();       allow create: if isCreatingOwnDoc();     }      match /creativeAssets/{assetId} {       allow read, update, delete: if isOwner();       allow create: if isCreatingOwnDoc();     }      match /creativeEdges/{edgeId} {       allow read, update, delete: if isOwner();       allow create: if isCreatingOwnDoc();     }      match /creativeMetrics/{metricId} {       allow read, update, delete: if isOwner();       allow create: if isCreatingOwnDoc();     }      match /activityLogs/{activityId} {       allow read: if isOwner();       allow create: if isCreatingOwnDoc();       allow update, delete: if false;     }   } }
6.3 Storage Rules Draft
Recommended storage path:
users/{userId}/products/{productId}/creatives/{creativeId}/assets/{fileName}
Draft rules:
rules_version = '2';  service firebase.storage {   match /b/{bucket}/o {     function isSignedIn() {       return request.auth != null;     }      function isOwner(userId) {       return isSignedIn() && request.auth.uid == userId;     }      function isAllowedImage() {       return request.resource.contentType.matches('image/.*')         && request.resource.size < 10 * 1024 * 1024;     }      function isAllowedVideo() {       return request.resource.contentType.matches('video/.*')         && request.resource.size < 100 * 1024 * 1024;     }      match /users/{userId}/products/{productId}/creatives/{creativeId}/assets/{fileName} {       allow read: if isOwner(userId);       allow write: if isOwner(userId) && (isAllowedImage() || isAllowedVideo());       allow delete: if isOwner(userId);     }      match /users/{userId}/products/{productId}/exports/{fileName} {       allow read, write, delete: if isOwner(userId);     }   } }

7. Core Workflows
7.1 Create Product
Input:
type CreateProductInput = {   name: string   description?: string   niche?: string }
Process:
	•	Validate user session.
	•	Validate input.
	•	Create product document.
	•	Set userId from auth user.
	•	Set status active.
	•	Set timestamps.
	•	Create activity log.
Output:
Product

7.2 Create Test Batch
Input:
type CreateBatchInput = {   productId: string   name: string   objective?: string   sourceCreativeId?: string   startedAt?: Date }
Process:
	•	Validate user owns product.
	•	Query latest batch for product.
	•	Set batchNumber = latest + 1.
	•	If sourceCreativeId exists, validate creative is winner.
	•	Create batch.
	•	Create activity log.
Output:
TestBatch

7.3 Add Creative to Batch
Input:
type CreateCreativeInput = {   productId: string   batchId: string   parentCreativeId?: string    hook: string   primaryText?: string   headline?: string   angle?: string   imagePrompt?: string   videoPrompt?: string   format: "image" | "video" | "carousel" }
Process:
	•	Validate user owns product and batch.
	•	Generate creativeNumber.
	•	If parentCreativeId exists, fetch parent creative.
	•	Create dnaSnapshot from parent creative.
	•	Create creative document.
	•	If parentCreativeId exists, create creative edge.
	•	Create activity log.
Output:
Creative

7.4 Upload Asset

	•	Validate file type.
	•	Validate file size.
	•	Generate storage path.
	•	Upload file to Cloud Storage.
	•	Create creativeAssets document in Firestore.
	•	Create activity log.

users/{userId}/products/{productId}/creatives/{creativeId}/assets/{timestamp}-{fileName}

CreativeAsset

Input:

14.5 Layout Algorithm MVP
function buildCanvasLayout(batches, creatives, edges) {   const nodes = []    batches.forEach((batch, batchIndex) => {     const batchCreatives = creatives.filter(       creative => creative.batchId === batch.id     )      batchCreatives.forEach((creative, creativeIndex) => {       nodes.push({         id: creative.id,         type: "creativeNode",         position: {           x: batchIndex * 420,           y: creativeIndex * 180,         },         data: mapCreativeToNodeData(creative),       })     })   })    return {     nodes,     edges: edges.map(mapCreativeEdgeToFlowEdge),   } }
