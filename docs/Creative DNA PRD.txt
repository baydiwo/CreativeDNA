PRD: Creative DNA
1. Product Overview
1.1 Nama Produk
Creative DNA
1.2 Deskripsi Singkat
Creative DNA adalah aplikasi web untuk mencatat, mengelola, dan melacak perkembangan creative iklan dari satu produk ke produk lainnya.
Aplikasi ini dibuat untuk advertiser, media buyer, dan creative strategist yang melakukan creative testing di Meta Ads. Creative DNA membantu user melihat creative mana yang menang, creative mana yang menjadi sumber horizontal scaling, dan bagaimana hubungan antar creative terbentuk dari satu test batch ke test batch berikutnya.
Creative DNA bukan hanya tempat menyimpan gambar dan copy iklan. Creative DNA adalah sistem tracking creative lineage yang menyimpan “DNA” dari setiap creative:
	•	Angle
	•	Hook
	•	Primary text
	•	Headline
	•	CTA
	•	Prompt gambar
	•	Prompt video
	•	Visual notes
	•	Asset gambar/video
	•	Metrics
	•	Winner decision
	•	Parent creative
	•	Child creative
1.3 Platform Direction
Creative DNA akan dibangun sebagai aplikasi web berbasis:
	•	Next.js
	•	Firebase Authentication
	•	Cloud Firestore
	•	Cloud Storage for Firebase
	•	Firebase Hosting atau Firebase App Hosting
	•	React Flow / xyflow untuk canvas lineage
Firebase Studio dapat digunakan sebagai alat prototyping awal jika workspace masih tersedia. Namun, fondasi production tetap menggunakan Firebase services dan source code yang bisa dipindahkan ke repo mandiri.
1.4 Problem Statement
Advertiser sering membuat banyak creative untuk satu produk. Dalam satu test campaign, user bisa menguji 2 sampai 12 creative. Setelah 3 sampai 5 hari, user menandai creative tertentu sebagai winner.
Masalah muncul ketika winner tersebut dipakai untuk horizontal scaling. Hook, primary text, prompt gambar, angle, visual style, dan alasan kenapa creative itu menang sering tercecer di banyak tempat:
	•	Google Sheets
	•	Folder asset
	•	Chat
	•	Notion
	•	Meta Ads Manager
	•	Memory pribadi
Akibatnya:
	•	Sulit melakukan tracing dari creative winning ke creative turunan.
	•	Sulit tahu angle mana yang benar-benar diterima market.
	•	Sulit tahu apakah creative menang karena hook, visual, prompt, format, atau kombinasi semuanya.
	•	Sulit membuat creative baru yang tetap mempertahankan DNA winner sebelumnya.
	•	Scaling creative jadi tidak sistematis.
	•	Setelah beberapa ronde testing, user kehilangan peta hubungan antar creative.
1.5 Product Vision
Menjadi pusat kendali creative testing dan horizontal scaling yang membantu advertiser membaca silsilah performa dari setiap creative.
Creative DNA membantu user menjawab:
	•	Creative mana yang menang?
	•	Creative itu berasal dari creative mana?
	•	Hook apa yang dipertahankan?
	•	Visual apa yang diubah?
	•	Prompt gambar mana yang melahirkan winner?
	•	Angle mana yang paling sering diterima market?
	•	Garis keturunan creative mana yang paling profitable?
1.6 Target User
Primary User
Solo advertiser, digital marketer, founder, atau performance marketer yang menjalankan Meta Ads untuk produk sendiri.
Karakteristik:
	•	Rutin membuat banyak creative iklan.
	•	Melakukan testing manual 2 sampai 12 creative per batch.
	•	Melakukan horizontal scaling dari creative winning.
	•	Belum membutuhkan integrasi Meta API di tahap MVP.
	•	Membutuhkan sistem pencatatan creative yang rapi.
Secondary User
Media buyer, creative strategist, dan small agency.
Karakteristik:
	•	Mengelola beberapa produk atau brand.
	•	Membutuhkan riwayat creative untuk analisis internal.
	•	Ingin melihat angle dan hook yang terbukti diterima market.
	•	Membutuhkan canvas visual agar alur creative mudah dibaca.
1.7 Core Use Case
User membuat produk baru, lalu membuat Test Campaign 1. Di dalam campaign tersebut, user menginput 2 sampai 12 creative yang akan dites di Meta Ads.
Setelah 3 sampai 5 hari, user kembali ke aplikasi, input metrics secara manual, lalu menandai Creative 4 sebagai winner.
User kemudian membuat Test Campaign 2 dari Creative 4. Di Test Campaign 2, user menginput creative baru yang merupakan turunan dari Creative 4.
Setelah testing berikutnya, user menandai Creative 2 dari Test Campaign 2 sebagai winner.
Di halaman canvas, user bisa melihat garis lineage:
Creative 4 dari Test Campaign 1 → Creative 2 dari Test Campaign 2 → Creative winner berikutnya.
1.8 Product Goals
	•	Memudahkan user mencatat semua creative berdasarkan produk.
	•	Memudahkan user menandai creative winning.
	•	Memudahkan user membuat batch scaling dari creative winner.
	•	Memudahkan user melihat alur creative secara visual.
	•	Memudahkan user membaca angle, hook, prompt, dan visual yang diterima market.
	•	Mengurangi kehilangan konteks saat horizontal scaling.
	•	Membuat workflow creative testing lebih sistematis.
1.9 Business Goals
	•	Membuat aplikasi internal untuk workflow Meta Ads.
	•	Membuka peluang menjadi SaaS sederhana untuk advertiser dan media buyer.
	•	Membangun database creative intelligence.
	•	Mempersiapkan fondasi untuk AI creative generation dan Meta API integration di masa depan.
1.10 Non-Goals untuk MVP
Fitur berikut tidak masuk MVP:
	•	AI generate creative otomatis.
	•	Integrasi langsung dengan Meta Ads API.
	•	Auto-sync metrics dari Meta.
	•	AI scoring creative.
	•	Creative fatigue prediction.
	•	Multi-user approval workflow.
	•	Billing SaaS.
	•	Public creative marketplace.
	•	Advanced attribution.
	•	A/B significance calculator.
Fokus MVP adalah pencatatan, tracing, lineage, dan visualisasi creative.

2. Key Concepts
2.1 Product
Product adalah produk atau offer yang sedang diiklankan.
Contoh:
	•	The Queen’s Touch
	•	The Royal Service
	•	BicaraManjaa
	•	Household OS
Setiap product memiliki test batch, creative, winner, asset, metrics, dan canvas lineage sendiri.
2.2 Test Batch
Test Batch adalah satu ronde pengujian creative.
Contoh:
	•	Test Campaign 1
	•	Test Campaign 2
	•	Scale Batch 1
	•	Hook Test Batch
	•	Visual Test Batch
Satu test batch biasanya berisi 2 sampai 12 creative.
2.3 Creative
Creative adalah satu unit iklan.
Creative terdiri dari:
	•	Copy
	•	Hook
	•	Angle
	•	Prompt
	•	Visual asset
	•	Format
	•	Metrics
	•	Status
	•	Notes
Creative bisa berupa:
	•	Image
	•	Video
	•	Carousel
2.4 Creative DNA
Creative DNA adalah data inti dari sebuah creative.
Isi Creative DNA:
	•	Angle
	•	Hook
	•	Primary text
	•	Headline
	•	Description
	•	CTA
	•	Image prompt
	•	Video prompt
	•	Visual notes
	•	Format
	•	Aspect ratio
	•	Placement
	•	Asset
	•	Metrics
	•	Winner reason
	•	Mutation type
2.5 Winner
Winner adalah creative yang ditandai user sebagai creative terbaik atau paling layak dipakai untuk scaling.
Winner bisa dipilih berdasarkan:
	•	CPA terbaik
	•	Purchase terbanyak
	•	ROAS terbaik
	•	CTR tinggi
	•	CPC rendah
	•	Visual paling relatable
	•	Hook paling kuat
	•	Manual judgment
2.6 Source Creative
Source Creative adalah creative winner yang menjadi sumber batch baru.
Contoh:
Creative 4 dari Test Campaign 1 menjadi source untuk Test Campaign 2.
2.7 Child Creative
Child Creative adalah creative baru yang dibuat dari source creative.
Contoh:
Creative 2 dari Test Campaign 2 adalah child dari Creative 4 di Test Campaign 1.
2.8 Lineage
Lineage adalah hubungan silsilah antar creative.
Contoh:
Creative 4 → Creative 2 → Creative 7
Lineage membantu user melihat cabang creative yang terus menghasilkan winner.
2.9 Mutation
Mutation adalah perubahan yang dilakukan dari parent creative ke child creative.
Contoh mutation:
	•	Same hook, new visual.
	•	Same visual, new hook.
	•	Same angle, new prompt.
	•	Same copy, new format.
	•	Same prompt, different model.
	•	Same headline, new primary text.

3. User Journey
3.1 First-Time User Journey
	•	User register atau login.
	•	User masuk ke dashboard.
	•	User klik “Create Product”.
	•	User mengisi nama produk.
	•	User masuk ke Product Workspace.
	•	User klik “Create Test Batch”.
	•	User menginput nama batch, objective, dan tanggal mulai.
	•	User menambahkan 2 sampai 12 creative.
	•	User upload asset gambar/video.
	•	User menyimpan batch.
	•	User menjalankan iklan di Meta secara manual.
	•	Setelah 3 sampai 5 hari, user kembali ke aplikasi.
	•	User input metrics manual.
	•	User menandai creative winner.
	•	User membuat scale batch dari winner.
	•	User melihat garis lineage di canvas.
3.2 Returning User Journey
	•	User login.
	•	User melihat daftar produk.
	•	User memilih produk.
	•	User membuka Lineage Canvas.
	•	User melihat creative winner terakhir.
	•	User klik winner.
	•	User membaca DNA creative.
	•	User membuat batch baru dari winner.
	•	User input creative turunan.
	•	User menjalankan testing lagi.

4.8 Create Scale Batch from Winner
User dapat membuat batch baru dari creative winner.
Flow
	•	User membuka creative winner.
	•	User klik “Create Scale Batch”.
	•	Sistem membuka form batch baru.
	•	Sistem mengisi sourceCreativeId dari winner.
	•	User mengisi nama batch dan objective.
	•	User membuat creative turunan.
	•	Setiap creative turunan otomatis punya parentCreativeId.
	•	Sistem membuat edge dari parent ke child.
Acceptance Criteria
	•	Hanya creative winner yang bisa menjadi source scale batch.
	•	Batch baru menyimpan sourceCreativeId.
	•	Creative baru di batch tersebut menyimpan parentCreativeId.
	•	Edge otomatis dibuat dari parent ke child.
	•	Canvas langsung memperlihatkan hubungan parent-child.

6. UX Requirements
6.1 Design Direction
Creative DNA harus terasa seperti creative command center.
Karakter UI:
	•	Clean
	•	Fast
	•	Visual
	•	Tactical
	•	Tidak terlalu enterprise
	•	Tidak terlalu spreadsheet
	•	Fokus ke creative card dan canvas
6.2 Suggested Visual Style
	•	Background: dark slate atau soft neutral.
	•	Card: clean white atau dark card.
	•	Accent: electric blue, purple, atau gold.
	•	Winner badge: green atau gold.
	•	Failed branch: muted red.
	•	Canvas edge: subtle.
	•	Typography: modern sans-serif.
6.3 Empty State
Dashboard kosong:
“Belum ada produk. Mulai dari satu produk dulu, lalu biarkan creative-nya punya silsilah.”
Canvas kosong:
“Belum ada lineage. Tandai creative sebagai winner, lalu buat scale batch dari creative tersebut.”
Winner kosong:
“Belum ada winner. Setelah campaign berjalan 3 sampai 5 hari, tandai creative yang layak jadi sumber scaling.”
6.4 Error State
	•	Upload gagal.
	•	Data gagal disimpan.
	•	Creative tidak ditemukan.
	•	User tidak punya akses.
	•	File terlalu besar.
	•	Format file tidak didukung.
6.5 Confirmation Modal
Dibutuhkan untuk:
	•	Delete product
	•	Delete batch
	•	Delete creative
	•	Remove winner status
	•	Archive product
	•	Create scale batch

10. Platform Risk
Firebase Studio dapat digunakan untuk prototyping, tapi tidak boleh menjadi satu-satunya tempat source code disimpan.
Mitigation:
	•	Simpan project di GitHub.
	•	Gunakan Firebase services sebagai backend utama.
	•	Pastikan app bisa dijalankan secara lokal.
	•	Pastikan deployment bisa dilakukan dari repo.
	•	Hindari lock-in ke fitur eksperimental Firebase Studio.
	•	Siapkan migrasi ke Google AI Studio, Google Antigravity, atau local development jika diperlukan.

11. Product Principle
Creative DNA harus membantu user menjaga tiga hal:
	•	Apa yang dites.
	•	Apa yang menang.
	•	Dari mana winner berikutnya berasal.
Jika sebuah fitur tidak membantu salah satu dari tiga hal ini, fitur tersebut tidak masuk MVP.
